# HNVNS Healthcare Staffing Marketplace

Production-ready dark minimal website for HNVNS, a healthcare staffing marketplace focused on imaging, diagnostics, and verified candidate matching.

## Tech stack

- Next.js 14 App Router
- TypeScript
- Tailwind CSS
- GSAP + ScrollTrigger
- Lenis smooth scrolling
- Framer Motion page transitions
- Inter font via `next/font`

## Project structure

```txt
app/                  Next.js routes and global styles
components/           Reusable UI, site, section, animation, and form components
lib/                  Shared data and utility functions
providers/            Client-side providers such as Lenis
public/               Static assets, including favicon placeholder
```

## Setup

```bash
npm install
npm run dev
```

Open http://localhost:5500

## Deploy to Vercel

### Option 1 (recommended)

- Go to vercel.com → New Project → Import Git repo
- Connect the repository
- Keep the default Next.js build settings
- Deploy

### Option 2 (quick)

```bash
npm i -g vercel
vercel login
vercel --prod
```

## Production environment variables

Add these variables in the Vercel project dashboard, or copy `.env.example` to `.env.local` for local development:

```txt
DATABASE_URL=Connection string to your Supabase/PostgreSQL database.
RESEND_API_KEY=API key for email routing.
BLOB_READ_WRITE_TOKEN=Storage credentials generated by adding Vercel Blob to the project in Vercel settings.
ADMIN_PASSCODE=Passcode for the admin dashboard login.
ADMIN_JWT_SECRET=Secret used to sign admin & dashboard session cookies (falls back to ADMIN_PASSCODE if unset).
UPLOAD_TOKEN=Static token required on every resume upload (sent as the X-Upload-Token header).
```

These activate the live PostgreSQL database, Resend email notifications, Vercel Blob resume storage, signed session cookies for the admin and candidate/hospital dashboards, and authenticated resume uploads.

### Security notes

- **Admin dashboard** (`/admin`): login uses a timing-safe passcode comparison and issues a signed httpOnly cookie (`hnvns_admin`, 4-hour expiry). All admin actions are gated by this session — the passcode is never re-sent.
- **Dashboard** (`/dashboard`): uses a 2-step email OTP flow. A 6-digit code (SHA-256 hashed at rest) is emailed via Resend; verifying it issues a signed httpOnly cookie (`hnvns_dashboard`, 2-hour expiry) bound to the user's email.
- **Resume uploads** (`/api/upload`): require a valid `X-Upload-Token` header, enforce 5 uploads/hour/IP, validate the `Content-Type` and `%PDF-1.` magic bytes, and store files under a UUID path.

## Customization Guide

- Update job data: `/lib/data.ts`
- Update colors: `tailwind.config.ts`
- Update copy: each `/app/*/page.tsx`
- Add real form backend: configure PostgreSQL, Resend, and Vercel Blob environment variables.

## Implemented features

- Sticky navigation with transparent-to-blurred scroll state
- Home, Services, About, Insights, Contact, Jobs, Hospitals, and Candidates pages
- Hero word-by-word GSAP reveal
- CSS infinite ticker animation
- GSAP ScrollTrigger section fade-ups
- Scroll-triggered stats count-up
- Lenis smooth scrolling
- Framer Motion page transitions
- Dark minimal design with teal accents and hover border glow
- Server-backed hospital vacancy and candidate registration flows with PostgreSQL, Resend, and Vercel Blob support
- Timing-safe admin passcode login with signed httpOnly session cookies
- Email OTP login for the candidate/hospital dashboard with hashed, expiring codes
- Hardened resume upload route (token auth, IP rate limiting, strict PDF validation, UUID storage paths)
